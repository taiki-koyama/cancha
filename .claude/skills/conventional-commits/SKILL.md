# コミットメッセージ規則（Conventional Commits）

## フォーマット

```
<type>: <日本語の説明>

[任意の本文]
```

## タイプ一覧

| type | 用途 |
|------|------|
| `feat` | 新機能の追加 |
| `fix` | バグ修正 |
| `refactor` | リファクタリング（機能変更・バグ修正なし） |
| `test` | テストの追加・修正 |
| `chore` | ビルド・CI・依存関係などの雑務 |
| `docs` | ドキュメントのみの変更 |
| `style` | コードスタイルの修正（動作変更なし） |
| `perf` | パフォーマンス改善 |

## ルール

- **説明は日本語**で書く
- 1行目は50文字以内に収める
- 現在形・命令形で書く（「〜を追加」「〜を修正」）
- 末尾にピリオドをつけない
- 破壊的変更がある場合は `feat!:` または `fix!:` とする

## 例

```
feat: Bedrock チャット機能を追加
fix: IME変換中のEnterキーで送信されるバグを修正
refactor: チャットハンドラーを関数に切り出し
test: /api/chat エンドポイントのテストを追加
chore: boto3 を requirements.txt に追加
docs: API エンドポイントの仕様をコメントに追記
```
