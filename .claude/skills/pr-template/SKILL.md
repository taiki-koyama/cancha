# PR テンプレート

## フォーマット

```markdown
## やったこと
- （箇条書きで変更内容を記載）

## やらなかったこと
- （スコープ外にしたこと、次回対応にしたことを記載）
- なし

## その他
- （レビュアーへの補足、動作確認方法、スクリーンショットなど）
- なし
```

## ルール

- **タイトル**は Conventional Commits 形式（`feat: 〜` / `fix: 〜`）
- 「やったこと」は **Why（なぜ）** も添える
- 「やらなかったこと」は意図的に除外した内容を明示する（レビュアーの「これは？」を防ぐ）
- 動作確認が必要な変更はスクリーンショットや手順を「その他」に記載

## タイトル例

```
feat: Bedrock チャット機能を追加
fix: IME変換中のEnterキーで送信されるバグを修正
chore: boto3 を requirements.txt に追加・pytest テスト追加
```
